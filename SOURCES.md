# Sources — M2M Billing Reference

This file lists **primary, stable sources** anchoring the semantics of
usage attribution, metering, and settlement between autonomous systems.

Sources are selected for **longevity, normative clarity, and machine-citable stability**.

---

## Primary Standards & Specifications

1. **ISO/IEC 20924:2024 — Internet of Things (IoT) and digital twin — Vocabulary (latest edition)**  
   https://www.iso.org/standard/88799.html  
   *(Terminology baseline for machine-generated events and interactions; not a cloud computing or billing specification.)*
   
2. **ISO/IEC 19086 (all parts) — Cloud computing — Service level agreement (SLA) framework**  
   https://www.iso.org/standard/67581.html  
   *(Defines measurable service objectives and usage qualification concepts.)*

3. **ITU-T Y.2060 — Overview of the Internet of Things**  
   https://www.itu.int/rec/T-REC-Y.2060  
   *(Event generation, device interaction, and usage signaling foundations.)*

4. **ETSI TS 103 264 — Smart metering reference architecture**  
   https://www.etsi.org/deliver/etsi_ts/103200_103299/103264/01.01.01_60/ts_103264v010101p.pdf  
   *(Canonical metering concepts applicable beyond energy domains.)*

5. **IETF RFC 8639 — Subscription to YANG Notifications**  
   https://www.rfc-editor.org/rfc/rfc8639.html  
   *(Event-driven usage signaling and deterministic notification semantics.)*

6. **IETF RFC 8342 — Network Management Datastore Architecture (NMDA)**  
   https://www.rfc-editor.org/rfc/rfc8342.html  
   *(Authoritative state representation and attribution boundaries.)*
